什么是TestNG

TestNG 是一个受 JUnit 和 NUnit 启发的测试框架，但引入了许多新功能，使其更强大且更易于使用。它被设计用于单元测试、集成测试和端到端测试。

使用方法

单个用例

单个用例情况下，选中并右键点击被@Test注解的函数,Run As ->TestNG Test

单个类的用例

未选中任何函数情况下，于要运行的类空白处右键，Run As ->TestNG Test，会执行该类下所有的@Test注解函数用例

TestNG 注解详解

TestNG 提供了丰富的注解来支持各种测试场景，下面将分类详细介绍这些注解及其用途。

核心测试注解

@Test

标记一个方法为测试方法。

常用属性：

  • enabled: 是否启用该测试（默认 true）
  • groups: 测试所属分组
  • dependsOnMethods: 依赖的其他测试方法
  • priority: 测试优先级（数值越小优先级越高）
  • timeOut: 超时时间（毫秒）
  • expectedExceptions: 期望抛出的异常
  • invocationCount: 调用次数
  • successPercentage: 成功百分比
  • dataProvider: 关联的数据提供者
  • alwaysRun: 是否总是运行（即使依赖的方法失败）

配置方法注解（生命周期钩子）

@BeforeSuite / @AfterSuite
  • @BeforeSuite: 在当前测试套件所有测试执行之前运行
  • @AfterSuite: 在当前测试套件所有测试执行之后运行
@BeforeTest / @AfterTest
  • @BeforeTest: 在 <test> 标签内的所有测试方法之前执行
  • @AfterTest: 在 <test> 标签内的所有测试方法之后执行
@BeforeClass / @AfterClass
  • @BeforeClass: 在当前类的第一个测试方法之前执行
  • @AfterClass: 在当前类的所有测试方法之后执行
@BeforeMethod / @AfterMethod
  • @BeforeMethod: 在每个测试方法之前执行
  • @AfterMethod: 在每个测试方法之后执行
@BeforeGroups / @AfterGroups
  • @BeforeGroups: 在指定分组的所有测试方法之前执行
  • @AfterGroups: 在指定分组的所有测试方法之后执行

参数化测试注解

@Parameters

从 testng.xml 文件中获取参数。

@DataProvider

提供测试数据。

常用属性：

  • name: 数据提供者名称
  • parallel: 是否并行运行（true / false）

依赖和分组注解

@Listeners

定义测试监听器。

@Factory

标记一个方法为工厂方法，用于动态创建测试实例。

五、其他实用注解

@Ignore / @Test(enabled=false)

跳过测试。

@NoInjection

防止 TestNG 自动注入参数。

suite.xml 语法以及与注解的交互详解

TestNG 提供了一种结构化的配置方式——通过 testng.xml（也称 suite.xml）文件来定义测试套件的执行策略。这个 XML 配置文件与 TestNG 的注解紧密协作，形成灵活而强大的测试执行流程。

testng.xml 基础语法结构

关键结构说明：
标签描述
<suite>表示一个测试套件，可以包含多个 <test>。支持并发设置，作用于全局测试环境。
<test>表示一个测试块，通常对应一个模块或功能组，内部可包含多个类或方法。
<parameter>用于为测试类或测试方法注入参数，与 @Parameters 注解配合使用。
<classes>用于指定当前 <test> 包含的测试类。
<class>指定一个测试类的完全限定名。
<methods>精确指定要运行的测试方法（或排除某些方法）。
<groups>控制分组执行行为，配合 @Test(groups = "...") 使用。
<listeners>注册监听器类，支持日志、报告、自定义行为的扩展。

testng.xml 与注解的详细交互

XML 标签关联注解作用范围触发时机推荐用途
<suite>@BeforeSuite, @AfterSuite整个 XML 执行生命周期（全局初始化与清理）开始和结束时各执行一次数据库连接池创建、日志系统初始化、环境配置等全局资源处理
<test>@BeforeTest, @AfterTest每个 <test> 区块（模块级）每个 <test> 标签开始/结束前后针对特定模块做准备，比如初始化模块配置、API Token 等
<class>@BeforeClass, @AfterClass每个测试类测试类中第一个方法执行前 / 全部方法执行后类级别的资源准备，如启动/关闭浏览器、创建 HTTP 客户端
<methods>@BeforeMethod, @AfterMethod每个测试方法每个方法前后各执行一次用于数据初始化、Mock、资源释放等方法级场景
<groups>@Test(groups = "..."), @BeforeGroups, @AfterGroups分组测试方法分组内所有方法执行前 / 后控制特定类别测试初始化，例如 database、ui、api 分组资源
<parameter>@Parameters方法参数注入方法执行时注入用于传递外部配置，如用户名、密码、URL、版本号等
<listeners>@Listeners（类注解替代）整个测试执行过程初始化时注册实现日志收集、截图、失败重试、报告生成等钩子行为
