介绍

Appium 是一个开源的 跨平台移动应用自动化测试工具，支持：
iOS（基于 XCUITest）
Android（基于 UIAutomator2/Espresso）
Windows 桌面应用（WinAppDriver）

ADB（Android Debug Bridge）工作原理详解

组件作用运行位置
ADB Client（客户端）发送命令（如 adb devices、adb install）你的 电脑（PC/Mac）
ADB Server（服务端）管理客户端与设备的通信，后台运行电脑（默认端口 5037）
ADB Daemon（adbd，守护进程）在设备端执行命令Android 设备或模拟器
安卓手机/模拟器实际运行命令的目标设备物理设备或虚拟环境

流程图:

    sequenceDiagram
    participant PC as 电脑（PC/Mac）
    participant Server as ADB Server
    participant Device as Android 设备
    PC ->> Server: 执行命令（如 `adb devices`）
    Server ->> Device: 通过 USB/WiFi 连接 adbd
    Device -->> Server: 返回设备列表
    Server -->> PC: 显示结果

关键步骤解析

  1. 客户端发起请求

    • 当你在终端输入 adb devices，ADB Client 会向本机的 ADB Server（端口 5037） 发送请求。

  2. 服务端管理连接

    • ADB Server 检查是否有设备连接（通过 USB 或网络）。
    • 如果设备已连接，Server 会与设备上的 adbd 守护进程通信。

  3. 设备端执行命令

    • adbd 在设备端接收命令（如列出设备、安装 APK），执行后返回结果。

  4. 结果返回客户端

    • Server 将设备端的响应传回 Client，最终显示在终端。

ADB 的通信方式

(1) USB 连接（默认方式）

  • 设备通过 USB 线连接电脑，需开启 USB 调试模式（开发者选项 → USB 调试）。
  • 底层协议：使用 USB 协议 传输数据。

(2) 网络连接（WiFi 调试）

  • 通过 TCP/IP 连接设备（无需 USB 线）：
  • 底层协议：基于 TCP/IP 协议。

元素操作API

元素定位函数

多元素定位区别是

  • 返回值是列表，查不到元素返回空
  • 单元素定位空返回NoSuchElementException错误
  • 函数是find_elements

以下以单元素为例

find_element(AppiumBy.ID, "com.example.myapp:id/submit_button")

  • 作用：通过元素的唯一标识符（resource-id）定位元素。
  • 参数来源：Android 应用中元素的 resource-id 属性，通常格式为 包名:id/元素ID。
  • 适用场景：推荐使用，因其唯一性高，定位速度快。

find_element(AppiumBy.CLASS_NAME, "android.widget.EditText")

  • 作用：通过元素的类名定位元素。
  • 参数来源：元素的类名，如 android.widget.EditText。
  • 适用场景：当元素的类名具有唯一性时使用，但由于类名可能重复，建议谨慎使用。

find_element(AppiumBy.TAG_NAME, "input")

  • 作用：通过元素的标签名定位元素。
  • 参数来源：HTML 元素的标签名，如 input。
  • 适用场景：主要用于 WebView 中的 HTML 元素定位。

find_element(AppiumBy.XPATH, "//android.widget.TextView[@text='登录']")

  • 作用：通过 XPath 表达式定位元素。
  • 参数来源：元素的属性组合，如类名、文本等。
  • 适用场景：适用于复杂或动态元素的定位，但性能较低，建议优先使用其他定位方式。

find_element(AppiumBy.CSS_SELECTOR, "input[type='text']")

  • 作用：通过 CSS 选择器定位元素。
  • 参数来源：HTML 元素的 CSS 属性。
  • 适用场景：仅适用于 WebView 中的 HTML 元素定位。

find_element(AppiumBy.LINK_TEXT, "点击这里")

  • 作用：通过链接的完整文本定位元素。
  • 参数来源：HTML <a> 标签的文本内容。
  • 适用场景：仅适用于 WebView 中的 HTML 链接定位。

find_element(AppiumBy.PARTIAL_LINK_TEXT, "点击")

  • 作用：通过链接的部分文本定位元素。
  • 参数来源：HTML <a> 标签的部分文本内容。
  • 适用场景：仅适用于 WebView 中的 HTML 链接定位。
