软件介绍

Apache JMeter 是一款开源的性能测试工具，主要用于测试 Web 应用程序的性能和负载能力。它最初由 Apache Software Foundation 开发，用于测试 Web 应用程序。

优缺点

优点

1、开源免费 2、跨平台
3、支持多协议 4、小巧

缺点

1、不支持IP欺骗
2、使用Jmeter无法验证JS程序，也也无法验证页面UI，要和Selenuim配合来完成Web测试

以下是 JMeter 中主要组件的功能说明、使用场景及常用函数的使用方法，按类别分类整理：

线程组（Thread Groups）

setUp 线程组

  • 作用：在正式测试前执行初始化操作（如登录、准备测试数据）。
  • 使用场景：预加载数据、建立数据库连接、生成 Token。
  • 特点：在所有其他线程组之前运行。

tearDown 线程组

  • 作用：在测试结束后执行清理操作（如删除临时数据、退出登录）。
  • 使用场景：释放资源、清理测试环境。
  • 特点：在所有其他线程组之后运行。

开放模型线程组（Open Model Thread Group）

  • 作用：通过自定义代码动态控制线程数（需配合 jsr223 脚本）。
  • 使用场景：模拟复杂的并发模型（如根据时间动态调整负载）。

普通线程组（Thread Group）

  • 作用：定义并发用户数、循环次数、调度策略。
  • 关键参数：
    • 线程数（虚拟用户数）
    • Ramp-Up 时间（逐步启动线程的时间）
    • 循环次数（或无限循环）

配置元件（Config Elements）

CSV 数据文件设置

  • 作用：从 CSV 文件读取参数（如用户名、密码）。
  • 使用场景：参数化测试数据。
  • 关键配置：
    • 文件名：CSV 文件路径。
    • 变量名：定义变量名（如 user,pass）。

HTTP 信息头管理器

  • 作用：添加 HTTP 请求头（如 Content-Type: application/json）。
  • 使用场景：模拟浏览器请求或 API 认证。

HTTP Cookie 管理器

  • 作用：自动管理会话 Cookie（如登录状态）。
  • 使用场景：需要保持会话的测试（如购物车流程）。

HTTP 缓存管理器

  • 作用：模拟浏览器缓存行为。
  • 使用场景：减少重复请求静态资源（如图片、JS）。

HTTP 请求默认值

  • 作用：设置所有 HTTP 请求的默认值（如服务器地址、端口）。
  • 使用场景：统一管理请求的基础配置。

用户定义的变量

  • 作用：定义全局变量（如 ${host}）。
  • 注意：变量在测试启动时初始化，运行时不可修改。

监听器（Listeners）

查看结果树（View Results Tree）

  • 作用：查看每个请求的详细请求/响应数据。
  • 注意：仅用于调试，性能测试时禁用（消耗内存）。

聚合报告（Aggregate Report）

  • 作用：生成关键性能指标（TPS、平均响应时间、错误率）。
  • 常用指标：
    • Throughput：每秒事务数（TPS）。
    • Average：平均响应时间。

后端监听器（Backend Listener）

  • 作用：将测试结果发送到外部监控系统（如 InfluxDB + Grafana）。

汇总报告（Summary Report）

  • 作用：简洁的统计结果表格（类似聚合报告）。

定时器（Timers）

固定定时器（Constant Timer）

  • 作用：在每个请求后添加固定延迟（如 1000 毫秒）。

同步定时器（Synchronizing Timer）

  • 作用：模拟瞬时并发（如秒杀场景）。
  • 关键参数：Number of Simulated Users to Group by。

高斯随机定时器（Gaussian Random Timer）

  • 作用：按正态分布随机延迟（更接近真实用户行为）。

前置/后置处理器（Pre/Post Processors）

正则表达式提取器

  • 作用：从响应中提取数据（如 Token）。
  • 示例：提取 "token":"(.*?)" 到变量 ${token}。

JSON 提取器

  • 作用：从 JSON 响应中提取字段（如 $.data.userId）。

用户参数（User Parameters）

  • 作用：为每个线程定义不同的变量值。

断言（Assertions）

响应断言

  • 作用：验证响应内容、状态码或响应时间。
  • 常用检查项：
    • 响应文本包含特定字符串。
    • 状态码等于 200。

JSON 断言

  • 作用：验证 JSON 结构或字段值（如 $.success == true）。

断言持续时间

  • 作用：检查响应时间是否超过阈值（如 500ms）。

常用函数（Functions）

JMeter 提供内置函数，通过 ${__functionName(arg1,arg2)} 调用：

  1. ${__Random(min,max)}

    • 生成随机数（如 ${__Random(1,100)}）。

  2. ${__time(format)}

    • 获取当前时间（如 ${__time(yyyy-MM-dd)}）。

  3. ${__threadNum}

    • 返回当前线程编号。

  4. ${__CSVRead(file.csv,0)}

    • 从 CSV 文件读取数据（替代 CSV 数据文件设置）。

  5. ${__P(propertyName)}

    • 读取 JMeter 属性（如 ${__P(threads)}）。

典型测试流程

  1. 配置测试计划：添加线程组、HTTP 请求默认值。
  2. 参数化数据：使用 CSV 数据文件或随机变量。
  3. 添加逻辑控制：循环控制器、If 控制器。
  4. 设置断言：验证响应正确性。
  5. 添加监听器：聚合报告、结果树（调试用）。
  6. 运行与分析：关注 TPS、响应时间、错误率。

注意事项

  1. 避免在正式测试时启用“查看结果树”（内存消耗大）。
  2. 参数化数据：使用 CSV 或 JDBC 连接数据库。
  3. 分布式测试：通过多台机器生成更高负载。
  4. 定时器作用域：定时器对其所在层级的 Sampler 生效。
